GitHub Pages Setup

اپنی files GitHub پر upload کریں

1
GitHub account بنائیں
github.com پر جائیں → Sign Up → مفت account بنائیں۔
2
New Repository بنائیں
Login کے بعد → "+ New" → Repository name: newsflow-site → Public → Create Repository
# Repository name ضروری ہے public ہو Repository name: newsflow-site Visibility: Public ← GitHub Pages کے لیے ضروری ✓ Add README file
3
Files Upload کریں
"Add file" → "Upload files" → چاروں files drag کریں → "Commit changes"
newsflow-site/ ├── index.html ← News Site (main page) ├── pipeline.html ← Content Pipeline ├── guide.html ← Admin Guide └── deploy.html ← یہ file
4
GitHub Pages Enable کریں
Repository → Settings → Pages → Source: "Deploy from a branch" → Branch: main → / (root) → Save
Settings → Pages → Source: Deploy from a branch Branch: main → Folder: / (root) → Save ✅ آپ کی site URL ہوگی: https://yourusername.github.io/newsflow-site/
5
Custom Domain لگائیں (اختیاری)
اگر آپ کا domain ہے تو: Settings → Pages → Custom Domain → اپنا domain ڈالیں۔ Domain registrar پر CNAME record بنائیں:
# Domain registrar DNS settings میں: Type: CNAME Name: www Value: yourusername.github.io # یا A records (apex domain کے لیے): 185.199.108.153 185.199.109.153 185.199.110.153 185.199.111.153
✅ بس! 2-3 منٹ میں آپ کی site live ہو جائے گی: https://yourusername.github.io/newsflow-site/
⚠ صرف GitHub Pages سے صرف آپ کا browser pipeline اور site کو connect کرے گا۔ دوسرے visitors کو pipeline posts نہیں ملیں گی۔ اس کے لیے Firebase بھی setup کریں۔
🔥

Firebase Realtime Database Setup

مفت online database — posts کہیں سے بھی accessible

1
Firebase Console کھولیں
console.firebase.google.com پر جائیں → Google account سے login کریں → "Create a project"
Project name: newsflow-cms Google Analytics: Disable (ابھی ضرورت نہیں) → Create Project → Continue
2
Realtime Database بنائیں
Left sidebar → Build → Realtime Database → "Create Database" → Start in test mode → Enable
# Test mode — 30 دن مفت access Rules: { "rules": { ".read": true, ← سب پڑھ سکتے ہیں ".write": true ← سب لکھ سکتے ہیں (30 دن) } }
⚠ 30 دن بعد rules update کرنے ہوں گے — نیچے دیکھیں
3
Database URL نوٹ کریں
Database بنانے کے بعد یہ URL ملے گی — اسے copy کریں:
# آپ کی Database URL کچھ ایسی ہوگی: https://newsflow-cms-default-rtdb.firebaseio.com/ # یا regional: https://newsflow-cms-default-rtdb.asia-southeast1.firebasedatabase.app/
4
Web App Register کریں
Project Overview → "</>" icon (Web) → App nickname: "newsflow" → Register App → Config copy کریں
// آپ کو یہ config ملے گا: const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "newsflow-cms.firebaseapp.com", databaseURL: "https://newsflow-cms-default-rtdb.firebaseio.com", projectId: "newsflow-cms", storageBucket: "newsflow-cms.appspot.com", messagingSenderId: "123456789", appId: "1:123:web:abc" };
5
30 دن بعد — Rules Update کریں
Realtime Database → Rules tab → یہ rules paste کریں تاکہ صرف admin لکھ سکے:
{ "rules": { "posts": { ".read": true, // کوئی بھی پڑھ سکتا ہے ".write": "auth != null" // صرف logged-in user لکھ سکتا ہے } } }

Code Changes

firebase.js بنائیں — پھر index.html اور pipeline.html میں add کریں

💡 صرف ایک نئی file بنانی ہے: firebase.js — اور دونوں HTML files میں ایک line add کرنی ہے۔ باقی سب کام یہ file کرے گی۔
firebase.js — نئی file بنائیں
// ===================================================== // firebase.js — NewsFlow CMS Shared Database Layer // ===================================================== // یہ file index.html اور pipeline.html دونوں میں // <script src="firebase.js"></script> سے add کریں // ===================================================== // ★ اپنا Firebase config یہاں ڈالیں const FIREBASE_CONFIG = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT.firebaseapp.com", databaseURL: "https://YOUR_PROJECT-default-rtdb.firebaseio.com", projectId: "YOUR_PROJECT_ID", storageBucket: "YOUR_PROJECT.appspot.com", messagingSenderId: "YOUR_SENDER_ID", appId: "YOUR_APP_ID" }; // Firebase SDK load کریں const _fbScript = document.createElement('script'); _fbScript.src = 'https://www.gstatic.com/firebasejs/9.23.0/firebase-app-compat.js'; _fbScript.onload = () => { const s2 = document.createElement('script'); s2.src = 'https://www.gstatic.com/firebasejs/9.23.0/firebase-database-compat.js'; s2.onload = initFirebase; document.head.appendChild(s2); }; document.head.appendChild(_fbScript); let db = null; let fbReady = false; function initFirebase() { firebase.initializeApp(FIREBASE_CONFIG); db = firebase.database(); fbReady = true; console.log('✅ Firebase connected'); // Dispatch event so page knows Firebase is ready window.dispatchEvent(new Event('firebase-ready')); } // ===================================================== // PUBLIC API — دونوں files یہ functions استعمال کریں // ===================================================== // Post save کریں (pipeline.html use کرے گی) async function fbSavePost(post) { if (!fbReady) { fallbackSave(post); return; } const ref = db.ref('posts/' + post.id); await ref.set({ ...post, updatedAt: Date.now() }); console.log('✅ Post saved to Firebase:', post.title); } // تمام posts لوڈ کریں (index.html use کرے گی) async function fbLoadPosts() { if (!fbReady) return fallbackLoad(); const snap = await db.ref('posts').orderByChild('updatedAt').once('value'); const data = snap.val() || {}; return Object.values(data).reverse(); } // Post delete کریں async function fbDeletePost(id) { if (!fbReady) return; await db.ref('posts/' + id).remove(); } // Real-time listener — نئی post آئے تو site خود refresh ہو function fbListenPosts(callback) { if (!fbReady) { setTimeout(() => fbListenPosts(callback), 1000); return; } db.ref('posts').on('value', snap => { const data = snap.val() || {}; callback(Object.values(data).reverse()); }); } // Fallback to localStorage اگر Firebase na chale function fallbackSave(post) { const posts = fallbackLoad(); const idx = posts.findIndex(p => p.id === post.id); if (idx >= 0) posts[idx] = post; else posts.unshift(post); localStorage.setItem('nf_fb_fallback', JSON.stringify(posts)); } function fallbackLoad() { try { return JSON.parse(localStorage.getItem('nf_fb_fallback') || '[]'); } catch(e) { return []; } }
